git的前期准备

下载git​

这里是下载的链接git download

🛠 如果你下载 Git 安装包，安装的到底是什么？ 当你安装 Git 的时候，安装包包含了以下几部分内容：

1️⃣ Git 命令行工具（核心部分）

2️⃣ Git 环境变量的配置

3️⃣ Git Bash（Windows 上的终端工具）

4️⃣ 一些 GUI 工具（可选）

可以根据自己的需求安装适配自己系统的安装包

这时候我们可以在vscode中的终端输入以下代码来查看是否成功的下载了

git --version

如果你得到了与下方图片相似的输出就说明安装成功了

前期的配置​

在下载后git后 我们需要一些简单的配置让接下来的使用中进行的更加的顺利

设置你的username和邮箱​

在使用 Git 进行版本控制时，Git 会记录每次提交的 用户名 和 邮箱地址。正确配置这些信息，可以确保你的提交记录和 GitHub/Gitee 账户绑定，从而显示你的头像和个人信息。因为我们推荐绑定自己账户注册的邮箱以及用户名

特殊email

如果你不想用自己的邮箱去创建他,我们还可以用github提供的邮箱 yourusername@users.noreply.github.com

这里是示范

git config --global user.name "linsen25"
git config --global user.email "linsen25@example.com"

现在我们可以通过git命令来看到我们设置的username以及邮箱

当然如果输入错误也不用担心只需要再执行一次设置就可以覆盖掉上一次的更改

git config --global --list

绑定你的github​

当你生成 SSH 密钥 并将它绑定到 GitHub 后，你的本地 Git 仓库和 GitHub 远程仓库之间可以通过 SSH 安全通道 进行通信。这种绑定关系给你带来了 更安全、更便捷的操作体验。

好处

在绑定了你的github之后你的上传 就不再每次收到密码认证了

🧩 步骤 1：生成 SSH 密钥​

1️⃣ 打开终端（macOS/Linux）或 Git Bash（Windows）。

2️⃣ 运行以下命令，生成 SSH 密钥：

ssh-keygen -t ed25519 -C "你的GitHub邮箱地址"

🔑 解释：

  • t ed25519：指定使用 ed25519 算法（更安全）。
  • C：添加注释，通常使用你的 GitHub 注册邮箱。

3️⃣ 生成之后他会为你默认生成一个保存的路径并且这个ssh是永久的除非你删除它

我们只需要Enter就可以确认这个决定

Generating public/private ed25519 key pair.
Enter file in which to save the key (/Users/yourname/.ssh/id_ed25519):

🧩 步骤 2：查看并复制 SSH 公钥​

1️⃣ 运行以下命令，查看 SSH 公钥：

cat ~/.ssh/id_ed25519.pub

2️⃣ 复制终端中输出的公钥，类似于以下格式：

ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIGK4R9C9QvCQK... linsen25@example.com

接下来我们看在实际中的效果

🧩 步骤 3：在 GitHub 上添加 SSH 公钥​

1️⃣ 登录 GitHub（https://github.com）。

2️⃣ 点击右上角头像 -> Settings。

3️⃣ 在左侧菜单中选择 SSH and GPG keys。

4️⃣ 点击 New SSH key。

5️⃣ Title：随意填写，比如 My MacBook SSH Key。

6️⃣ Key：粘贴刚才复制的公钥内容。

7️⃣ 点击 Add SSH key。

🧩 步骤 4：测试 SSH 连接​

1️⃣ 运行以下命令，测试 GitHub 的 SSH 连接：

ssh -T git@github.com

✅ 输出示例：

Hi linsen25! You've successfully authenticated, but GitHub does not provide shell access.

可喜可贺

你成功的开启了你的ssh只属于你的专属通道接下来开始学习如何发布以及其他功能叭～
