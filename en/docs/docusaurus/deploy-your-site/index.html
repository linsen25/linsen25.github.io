如何部署你的网站到github

我们选用最简单的方法，也就是github给的github desktop pc 软件来帮助我们发布到我们github上

这不是唯一途径

我们使用的github desktop并不是唯一途径 像命令行也可以但是难度偏高 因此我们选用github desktop更好的上手 接下来会以图文的形式来讲解

这路是下载的链接github desktop

自己的网站

github page是一个gihub提供的免费的提供给个人的网站他们的仓库名都是<username>.github.io这样的形式在接下来的教学中我们会一步一步一起来完成

创建github page repository​

正如我们所说他们都是repository只是名字不一样所以创建的步骤是一样的

首先登陆github点击->右上角的头像->点击your profile

然后在Tab栏中找到repository->点击右侧的new

然后输入<username>.github.io这样的形式直接创建

因为如果只是为了展示而不是开源就不需要license虽然这样说不太专业但是并不是强制的并且要选择public然后化到下面

重点

必须是public不能是private 如果只有自己能看到自己的网站那就没有意义了

创建好了这部分就算完成了～

更改docsaurus.config.js并定位你的目标仓库​

下面是一个表格说明了哪些是必须要改的在docsaurus的脚手架的基础上哪些是必须要改的title和tagline也是比较推荐更改的的

配置项说明示例值选填还是强制｜
title网站标题'My Project'选填
tagline网站标语'Documentation website'选填
urlGitHub Pages 访问地址'https://username.github.io'强制
baseUrl仓库名称路径'/my-project/'强制
organizationNameGitHub 用户名'username'强制
projectName仓库名称'my-project'强制
deploymentBranch部署分支'main'强制

现在我们直接展示代码例子

以我的账号的username为linsen25 我的repository name 是linsen25.github.io

docsaurus.config.js
module.exports = {
  title: 'My Personal Site',
  tagline: 'Docusaurus website',
  url: 'https://linsen25.github.io',    // GitHub Pages 访问地址
  baseUrl: '/',                         // 根路径，个人主页仓库必须是 '/'
  organizationName: 'linsen25',         // GitHub 用户名
  projectName: 'linsen25.github.io',    // 仓库名称
  deploymentBranch: 'main',             // 部署的分支
  trailingSlash: false,                 // 避免路径错误
  onBrokenLinks: 'throw',
  onBrokenMarkdownLinks: 'warn',
};

build 你的 docsaurus​

npm他是我们前面node.js包含的包管理器 以及npx都是包含在一起的

然在我们可以开始打包他不仅仅可以优化性能还可以更好的部署

现在打开我们的docsaurus的文件在VScode中,打开命令行工具的终端

然后在命令行中输入以下代码 然后回车便可得到一个名为build的文件夹

这个build文件就是我们要提交到我们的github pages上的文件

npm run build

github desktop上传build文件​

两种情况空的repository和不空

  • 如果repository有文件比如说README.md

如果repository是空的则点击左上方的Tab栏中的File选项选择clone到你的desktop

注意

如果是clone的话repository一定要是空的否则clone无效

  • 如果repository是已经有文件了

我们可以直接在左上角的current repository下拉框中直接点击然后选择

  • 提交

现在准备工作已经完成了

我们可以在Tab栏中选择repository然后open in vscode我们就可以看到现在我们这个仓库中包含的文件夹了

然后我们现在可以有两个面板一个是你的docsaurus的build文件一个是github desktop为我们打开的临时的vscode仓库中的文件。现在我们只需要将build文件中的文件全部复制到临时的vscode repository面板

现在我们回到github desktop我们会发现我们的repository被修改了

被修改的文件会出现在右边,左边是每次提交时可以留下一段话作为标记防止丢失以及提醒的作用现在我们点击commit to main

在commit之后我们可以看到左下角有个undo意味着我们可以撤销我们的commit 接下来是右上角的push to origin也是加上了一个任务可以被提交这时候我们点击他就成功更新你的repostory啦 同时你也可以开始访问你自己的网站了

恭喜你你已经成功的部署了属于自己的网站啦​

现在你只需要在google上搜索https://<username>.github.io

友情提示

要把username换成自己账号的名字比如说我的是linsen25 最后就是https://linsen25.github.io
